Creating symbolic links

It is inconvenient to copy all needed packages and modules into the application directory every time you develop a new appengine application. Implementing symbolic links that point from your application directory to a central location where reusable code is stored solves this problem. However, care must be taken as some of the packages can not be symlinked. The httplib2 package and the config.py file should not be symlinked. They will need to be copied directly into the application directory.

Example of directory structure for the gridmet_ppt_anomaly application

Note: "Google" is my directory for all things EE related. In my case, it is a sub directory of my home directory. The Google directory is where I store all reusable code (media, my-python-lib, ee-lib folders). Specific application code is stored in the applications folder. ~ stands for home directory and needs to be replaced with the appropriate path to your google directory. 

    ~/Google/
        ee-lib/
            ee/
            oauth2client/
            earthengine-api/
            pycrypto-2.6.1/
            google-api-python-client/
        my-python-lib/
            form_utils.py
            date_utils.py
            ee_apps.py
        media/
            img/
            css/
                google_style.css
            js/
                form_utils.js
                calendar.js
        applications/
            gridmet_ppt_anomaly/
                httplib2/
                ee -> ~/Google/ee-lib/ee
                media -> ~/Google/media
                date_urtils.py  -> ~/Google/my-python-lib/date_utils.py
                form_utils.py -> ~/Google/my-python-lib/form_utils.py
                config.py
                ee_appengine.py
                app.yaml
